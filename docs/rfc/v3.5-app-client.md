# v3.5 — App-client refactor

Status: Plan freeze (pending implement)
Owner: nghiahoang
Date: 2026-05-21

---

## Motivation

Current v3 client has these architectural smells:

1. **work = library**: clicking a manga forces an add-to-library write.
   Browse-only navigation doesn't exist; reading 1 chapter without
   commitment is impossible.
2. **work_id derived from `(source, upstream_ref)`**: tying server-side
   KV context to a source means changing sources = losing context.
   Multi-source linking needs stable nanoid ids.
3. **No prune**: Dexie can grow unbounded; offline storage budget UX
   missing.
4. **AddMangaModal is too tabbed**: current modal has 3 tabs (search /
   url / blank). Old design (commit `3095e48`) used a single-screen
   smart input that auto-detected URL vs text. Better UX.
5. **No history/work concept separation**: history is per-chapter only;
   "Continue reading" needs work-level last-opened tracking.

This refactor brings the client closer to native-app conventions
(Tachiyomi, Paperback, Komga) while preserving the v3 stateless server.

## Final architecture

### Dexie v3 schema

```ts
// Three primary stores. Everything else (jobs, archives, settings) unchanged.

works {
  id:                nanoid                       // PK, stable forever
  title:             string                       // user-editable
  cover_url:         string | null
  source_lang:       string
  target_lang:       string
  nsfw:              boolean

  sources: [{
    source:          string                       // adapter id (mdex, otruyen…)
    upstream_ref:    string                       // URL or stable id
    title:           string
    cover_url:       string | null
    languages:       string[]
    added_at:        ISO
  }]

  in_library:        boolean                      // user pinned this work
  library_status?:   'reading' | 'plan' | 'done' | 'dropped'
  library_added_at?: ISO

  last_opened_at:    ISO                          // any visit, browse-only OK

  created_at:        ISO
  updated_at:        ISO
  deleted?:          boolean                      // tombstone
}

history {
  id:                `${work_id}:${chapter_ref}`   // PK
  work_id:           string
  chapter_ref:       string                       // numberNorm
  chapter_label:     string                       // denormalized
  page:              number
  total_pages:       number | null
  last_read_at:      ISO
}

// archives, jobs, settings — no change.
```

### Indices

```ts
works:    '&id, in_library, last_opened_at, updated_at, *sourceKey'
history:  '&id, work_id, last_read_at'
```

`*sourceKey` is a multi-entry index: each entry in `sources[]` adds
`"${source}:${upstream_ref}"` to the index, enabling O(1) lookup
"given a (source, ref), find the work id".

### Resolve flow

```
useEnsureWorkFromSource(source, upstream_ref, snapshot) → work_id

  1. Lookup works where sourceKey = "${source}:${upstream_ref}"
  2. Hit → return existing id (update last_opened_at)
  3. Miss → create new work
       id            = nanoid()
       title         = snapshot.title
       cover_url     = snapshot.cover
       sources       = [{ source, upstream_ref, title, cover, languages, added_at }]
       in_library    = false                 // browse-only initially
       last_opened_at = now
     return new id
```

### Promote to library

```
useAddToLibrary(work_id, opts)
  works.update(work_id, {
    in_library: true,
    library_status: opts.status ?? 'reading',
    library_added_at: now,
    target_lang_override?: opts.target_lang,
  })

useRemoveFromLibrary(work_id)
  works.update(work_id, {
    in_library: false,
    library_status: null,
  })
  // Work still exists for browse-only; falls under prune later.
```

### Prune policy

```ts
const POLICY = {
  unpinnedWorks:     200,           // LRU by last_opened_at
  historyDays:       90,
  historyPerWorkMin: 5,              // always keep last 5 even if older
}

schedulePrune() on:
  - boot (5s delayed)
  - mutation idle (10s debounced)
```

Pinned (`in_library=true`) works never auto-drop.
Their context KV on the server is per-(user, work_id) with 180d TTL
refreshed on every job; no separate cleanup needed.

Restored from commit `3095e48` (legacy design), adapted to v3.5 schema.

### Detection logic

```ts
function detect(input: string): InputMode {
  if (/^https?:\/\//i.test(input.trim())) return 'url'
  if (input.trim().length < 2)             return 'empty'
  return 'search'
}
```

URL bar shows badge inline:
- `✓ Mangadex` (green) when host matches an installed source
- `⚠ Chưa hỗ trợ` (yellow) when no source owns this host

### URL mode

Auto-fetch detail from matched source. On success → show preview card +
"Thêm vào thư viện" button. Click → upsert work + library entry.

When host doesn't match → "Tạo trống thay" (blank create) fallback.

### Search mode

Fanout across enabled sources via `useFanoutSearch`. Scope tabs filter
client-side (no re-request). Per-source result cap (8) prevents one
chatty source from drowning others.

Each result row shows duplicate badge if a work already has this
material — clicking jumps to existing work instead of creating new.

Blank-create row at bottom: `[✚ Tạo trống "$query"]` — for offline-only
or upload-only works.

### Empty state

Query too short → show source roster as toggle chips for bulk
enable/disable. Helps user manage which sources participate in fanout.

## Routes

```
/                 Home — Continue reading + Recently added + Recent jobs
/library          Library — filter works where in_library=true
/explore          Explore — discover (multi-source browsing)
/w/$workId        Work hub — always available
                    if in_library  → full UI with chapters, context, history
                    else           → browse view with prominent "Thêm vào thư viện"
/r/$workId/$ch    Reader — chromeless
/jobs             Activity feed
/jobs/$id         Job detail
/settings         App settings
/login            Auth
/auth/callback    OAuth return
```

**10 routes total.** Reader + Login = chromeless. Rest = app shell.

`workId` is always nanoid. Source URLs never leak into routes.

### Browse-from-source flow

```
User clicks Mangadex card in /explore:
  1. useEnsureWorkFromSource(source: 'mdex', ref: '/title/op-uuid')
     → resolves or creates work_id (browse-only, in_library=false)
  2. router.navigate({ to: '/w/$workId', params: { workId } })
  3. Work hub renders with "Thêm vào thư viện" CTA
```

### Add to library from URL paste (modal)

```
User pastes URL in AddMangaModal:
  1. detect → 'url' mode
  2. host match → ✓ Mangadex
  3. fetchMangaDetail(manifest, url)
  4. useEnsureWorkFromSource(source, url, detail)
  5. useAddToLibrary(work_id)
  6. close modal + navigate to /w/$workId
```

### Multi-source attach

```
On Work hub, click "+ Thêm nguồn":
  Reuse the same fanout search component, scoped to "find this manga
  on another source". On pick:
  - Append to works[id].sources[]
  - update last_opened_at + updated_at
  - server context KV unchanged (work_id is stable)
```

## Hooks API (new)

```ts
// Work resolution + lifecycle
useWork(work_id)                        // dexie works.get
useWorkBySourceRef(source, ref)         // multi-entry index lookup
useEnsureWorkFromSource(source, ref, snapshot)  // get-or-create
useUpdateWorkSources(work_id, ...)      // add/remove source materials

// Library promotion
useLibraryWorks(filter?: { status })    // works where in_library=true
useAddToLibrary()
useRemoveFromLibrary()
useUpdateLibraryStatus()

// Reading state
useRecentlyOpened(limit)                // works ordered by last_opened_at
useContinueReading(limit)               // history join works, most recent
useRecordReading()                      // history upsert
useWorkHistory(work_id)

// Storage telemetry / prune
useStorageStats()                       // counts + byte estimates
useStorageActions()                     // { pruneNow, clearUnpinned, clearHistory }
```

## Success criteria

- Drop a zip → job appears in /jobs without library entry
- Browse Mangadex → click manga → /w/$workId opens, work not in library
- Click "Thêm vào thư viện" → work shows in /library
- Paste mangadex URL into modal → detail preview → 1-click add
- Open same manga from Otruyen → existing work attaches second source
- Settings storage panel shows "12 MB / 50 MB" with breakdown
- After 200 unpinned works visits, oldest are pruned
- All flows offline-capable except job creation (needs network)
